Nested Resampling

As we continually optimize over the same data during tuning or feature selection, the estimated performance of a learning algorithm might be optimistically biased. A clean approach to ensure unbiased performance estimation is nested resampling, where we embed the whole model selection process into an outer resampling loop.

The graphic illustrates nested resampling with 3-fold cross-validation in the outer and 4-fold cross-validation in the inner loop.

In the outer resampling loop, the data is split repeatedly into an outer training and an outer test set. In every outer iteration step, model selection is done by executing an inner resampling loop on the outer training set. Then the learner, using the selected hyperparameters and features, is fitted on the complete outer training set and its performance is evaluated on the outer test set.

%The resulting model is used to predict the target variable on the outer test set.

In mlr, we can get nested resampling for free without programming any looping by using the wrapper functionality. This works as follows:

  1. Generate a wrapped Learner via function makeTuneWrapper or makeFeatSelWrapper. Specify the inner resampling strategy using their resampling argument.
  2. Call function resample (see also the section about resampling) and pass the outer resampling strategy to its resampling argument.

Different inner and outer resampling strategies can be freely combined. The outer strategy can be a resample description (ResampleDesc) or a resample instance (ResampleInstance). The inner strategy should preferably be a ResampleDesc.

Tuning

As you might recall from the section about tuning, you need to define a search space by function makeParamSet, a search strategy by makeTuneControl*, and a method to evaluate hyperparameter settings (i.e., the inner resampling strategy and a performance measure).

Below is a classification example. We tune the cost parameter C and the RBF kernel parameter sigma of the support vector machine (ksvm) and use a grid search to find the hyperparameters with lowest error rate (mmce is the default measure for classification) on the inner test set (the inner resampling strategy is "Holdout"). The wrapped Learner is generated by calling makeTuneWrapper.

In the outer loop 5-fold cross-validation is used.

## Tuning in inner resampling loop
ps = makeParamSet(
  makeDiscreteParam("C", values = 2^(-2:2)),
  makeDiscreteParam("sigma", values = 2^(-2:2))
)
ctrl = makeTuneControlGrid()
inner = makeResampleDesc("Holdout")
lrn = makeTuneWrapper("classif.ksvm", resampling = inner, par.set = ps, control = ctrl, show.info = FALSE)

## Outer resampling loop
outer = makeResampleDesc("CV", iters = 5)
r = resample(lrn, iris.task, resampling = outer, extract = getTuneResult, show.info = FALSE)

Feature selection

As you might recall from the section about feature selection, mlr supports the filter and the wrapper approach.

Wrapper methods

Wrapper methods use the performance of a learning algorithm to assess the usefulness of a feature set. In order to select a feature subset a learner is trained repeatedly on different feature subsets and the subset which leads to the best learner performance is chosen.

For feature selection in the inner resampling loop, you need to choose a search strategy (function makeFeatSelControl*), a performance measure and the inner resampling strategy. Then use function makeFeatSelWrapper to bind everything together.

Below we use sequential forward selection with linear regression on the BostonHousing data set (bh.task). inner and outer resampling strategy.

## Feature selection in inner resampling loop
inner = makeResampleDesc("CV", iters = 3)
lrn = makeFeatSelWrapper("regr.lm", resampling = inner,
  control = makeFeatSelControlSequential(method = "sfs"), show.info = FALSE)

## Outer resampling loop
outer = makeResampleDesc("CV", iters = 3)
r = resample(learner = lrn, task = bh.task, resampling = outer, extract = getFeatSelResult,
  show.info = FALSE)

Filter methods with tuning

Filter methods assign an importance value to each feature. Based on these values you can select a feature subset by either taking all features with importance higher than a certain threshold or by taking a fixed number or percentage of the highest ranking features. Normally, neither the theshold nor the number or percentage of features is not known in advance and thus tuning is necessary.

In the example below the threshold value (fw.threshold) is tuned in the inner resampling loop. For this purpose the base Learner "regr.lm" is wrapped two times. First, makeFilterWrapper is used to fuse linear regression with a feature filtering preprocessing step. Then, a tuning step is added by makeTuneWrapper.

## Tuning of the percentage of selected filters in the inner loop
lrn = makeFilterWrapper(learner = "regr.lm", fw.method = "chi.squared")
ps = makeParamSet(makeDiscreteParam("fw.threshold", values = seq(0, 1, 0.1)))
inner = makeResampleDesc("CV", iters = 3)
lrn = makeTuneWrapper(lrn, resampling = inner, par.set = ps, control = ctrl, show.info = FALSE)

## Outer resampling loop
outer = makeResampleDesc("CV", iters = 3)
r = resample(learner = lrn, task = bh.task, resampling = outer, models = TRUE, show.info = FALSE)

Benchmark experiments

In a benchmark experiment multiple learners are compared on one or several tasks (see also the section about benchmarking). Nested resampling in benchmark experiments is achieved the same way as in resampling: First, use makeTuneWrapper or makeFeatSelWrapper to generate wrapped Learners with the inner resampling strategy of your choice. Second, call benchmark and specify the outer resampling strategies.

We will see four examples to show different benchmark settings:

  1. One data set + one classification algorithm + tuning
  2. Three data sets + two classification algorithms + tuning
  3. One data set + two regression algorithms + feature selection
  4. One data set + two regression algorithms + feature filtering + tuning

Example 1: One task, one learner, tuning

We show an example with outer bootstrap and inner cross-validation, our Learners are linear discriminant analysis and k-nearest neighbors. The number of neighbors k is tuned. choose accuracy instead of default measure mean misclassification error (mmce)

## Tuning of k-nearest neighbor in inner resampling loop
ps = makeParamSet(makeDiscreteParam("k", 1:5))
ctrl = makeTuneControlGrid()
inner = makeResampleDesc("CV", iters = 3)
lrn = makeTuneWrapper("classif.kknn", resampling = inner, par.set = ps, control = ctrl, show.info = FALSE)

## Learners
lrns = list(makeLearner("classif.lda"), lrn)

## Outer resampling loop
outer = makeResampleDesc("Bootstrap", iters = 5)
res = benchmark(lrns, iris.task, outer, measure = acc, show.info = FALSE)
res

Example 2: Two tasks, three learners, tuning

Now we look at an extensive example that shows a benchmark experiment with two data sets and three Learners. Two of the learners, ksvm and rpart, are tuned.

If you want to, you can use different inner resampling strategies for different learners. For example it might be practical to do fewer subsampling or bootstrap iterations for slower learners. (If you have larger benchmark experiments you might want to have a look at the section about parallelization.) Below we use FIXME for ksvm and FIXME for rpart.

You can also use different resampling strategies for different learning tasks by passing a list of resampling descriptions or instances to benchmark. For example if you have data sets of very different size, you might want to ... to minimize sample size bias. In the example below we use FIXME for the iris data set (iris.task) and FIXME for the sonar data (sonar.task).

## List of learning tasks
tasks = list(iris.task, sonar.task)

## Tune svm in the inner resampling loop
ps = makeParamSet(makeDiscreteParam("C", 2^seq(-1,1)),
  makeDiscreteParam("sigma", 2^seq(-1,1)))
ctrl = makeTuneControlGrid()
inner = makeResampleDesc("CV", iters = 3)
lrn1 = makeTuneWrapper("classif.ksvm", resampling = inner, par.set = ps, control = ctrl,
  show.info = FALSE)

## Tune rpart in the inner resampling loop
ps = makeParamSet(makeDiscreteParam("cp", c(0.05, 0.1, 0.2)))
ctrl = makeTuneControlGrid()
inner = makeResampleDesc("Holdout")
lrn2 = makeTuneWrapper("classif.rpart", resampling = inner, par.set = ps, control = ctrl,
  show.info = FALSE)

## Three learners to be compared
lrns = list(lrn1, lrn2, makeLearner("classif.lda"))

## Outer resampling loop
outer = list(makeResampleDesc("Holdout"), makeResampleDesc("CV", iters = 3))
res = benchmark(lrns, tasks, outer, measures = list(mmce, ber), show.info = FALSE)
res
